# PRD & Story Drafter

Turn a problem description into a structured PRD with user stories and
acceptance criteria — then pressure-test it with the questions a good PM
would ask.

Built by MBL Partners (https://mblpartners.com) — part of the Thrive with AI
skill library.

---

## What It Does

Takes your problem description and produces a complete PRD in three steps:

1. **Collect the core** — asks for the problem, the user, and what success
   looks like. One targeted follow-up if anything is vague.
2. **Draft the PRD** — produces a structured document with problem statement,
   goal and success metric, users and use cases, must-have / should-have /
   out-of-scope requirements, user stories with acceptance criteria, edge
   cases, and dependencies.
3. **Pressure-test it** — asks the 3–5 sharpest questions that surface what
   the draft missed while it is still cheap to fix, then folds the answers
   back in.

The format is solved. This skill handles the scaffolding so your time goes
to the product thinking.

## How to Run It

Once installed, trigger it by saying any of the following:
- "Draft a PRD"
- "Write the PRD"
- "Product requirements"
- "User stories for [feature]"
- "Spec this feature"

---

## Output

A complete PRD document containing:
- Problem statement (who has it, how often, what it costs today)
- Goal and measurable success metric
- Users and primary use cases
- Tiered requirements (must have / should have / out of scope)
- User stories with testable acceptance criteria
- Edge cases and open questions
- Dependencies and risks

Followed by 3–5 pressure-test questions that fold back into the draft.
